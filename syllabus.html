üìñ Course Description

This course provides a comprehensive introduction to fundamental data structures and algorithms essential for software development and computer science. Students will learn to analyze algorithm efficiency using Big O notation, implement classic data structures, and apply algorithmic problem-solving techniques.

The course covers linear data structures (arrays, linked lists, stacks, queues), non-linear structures (trees, graphs, heaps, hash tables), and fundamental algorithms for sorting, searching, and optimization. Emphasis is placed on understanding trade-offs between different approaches and selecting appropriate data structures for specific problems.

All implementations are done in Python, with a focus on both theoretical understanding and practical coding skills essential for technical interviews and real-world software development.

üéØ Learning Objectives

Upon successful completion of this course, students will be able to:

  1. Analyze algorithm complexity using Big O, Big Theta, and Big Omega notation
  2. Implement and manipulate fundamental data structures (arrays, linked lists, stacks, queues, trees, graphs, hash tables)
  3. Apply recursion and understand recursive problem decomposition
  4. Implement and analyze classic sorting algorithms (bubble, selection, insertion, merge, quick sort)
  5. Implement searching algorithms and understand their efficiency trade-offs
  6. Apply dynamic programming techniques to optimization problems
  7. Use greedy algorithms where appropriate and recognize their limitations
  8. Select appropriate data structures based on problem requirements and constraints
  9. Solve algorithmic problems commonly found in technical interviews

üìú Course Policies

Attendance

Attendance is mandatory. Students must attend at least 70% of classes to be eligible for the final exam. Weekly exercise submissions are tracked as part of attendance.

Late Submissions

Weekly exercises must be submitted during the designated submission window. Programming assignments have a 24-hour grace period with 20% penalty. After the grace period, submissions are not accepted.

Academic Integrity

All code submitted must be your own work. You may discuss approaches with classmates, but the implementation must be independent. Code similarity detection tools are used on all submissions. Using AI tools to generate solutions without understanding them is considered academic dishonesty.

Exam Policy

Exams include both written theory questions and live coding problems. No electronic devices are allowed during written portions. Coding portions are completed on lab computers with internet access disabled.

Office Hours

Office hours are available for discussing course material and getting help with assignments. Schedule appointments via email using your @istun.edu.tr address.
