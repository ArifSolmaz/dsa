How it works

📋 Workflow & submission guide

1
Open the notebook

Click an "Open in Colab" button above to open the notebook directly in Google Colab — no download needed. Alternatively, download the .ipynb file and open it in VS Code or Jupyter locally.

2
Read the project brief carefully

Each notebook starts with a scenario description, an input/output table, and conceptual background. Read this before writing any code — understanding the problem is half the solution.

3
Run the Setup cell first

The first code cell generates the dataset with a fixed random.seed. Everyone in the class gets the exact same data — this makes it possible to compare solutions and spot mistakes during the presentation.

4
Complete all four exercises

Each notebook has 4 exercises. Every exercise includes background theory, a precise task description, and tiered hints. Use them — they are there to guide you, not to spoil the problem. Uncomment the test lines at the bottom of each cell to check your output.

5
Submit via Blackboard before the deadline

Download your completed notebook as a .ipynb file (File → Download → Download .ipynb in Colab) and upload it to the corresponding assignment on Blackboard. Make sure all cells have been run and outputs are visible in the file before submitting.

6
Present live in the next class

On presentation day, you will be asked to open your notebook and walk through your solution — explaining what each function does, why you chose that approach, and what the output means. You may be asked to modify a small part on the spot. Presentations are short (3–5 min) but must show genuine understanding.

🤖 AI tools are allowed. Claude, Copilot, ChatGPT — use whatever helps you learn. However: during the live presentation, you must be able to explain every function and every line in your notebook. If you cannot explain code you submitted, it counts as zero regardless of correctness. The goal is understanding, not output.
